# ipcprims v0.2.1

**Release date**: 2026-04-04
**Lifecycle phase**: alpha
**License**: MIT OR Apache-2.0

## Summary

Windows named pipe transport (sync + async), full Windows CI/releng expansion, and developer experience improvements. This release replaces the Windows `Unavailable` placeholder with real named pipe implementations across all layers — transport, peer, and async — and validates them with CI jobs running on real Windows runners.

## Windows Named Pipe Transport

### Sync Transport

`ipcprims-transport` gains `NamedPipeStream` and `NamedPipeListener` replacing the `Unavailable` placeholder variant in `IpcStreamInner`:

| Type                | Purpose                                                                          |
| ------------------- | -------------------------------------------------------------------------------- |
| `NamedPipeStream`   | Sync read/write over Windows named pipes with overlapped I/O timeout enforcement |
| `NamedPipeListener` | Bind named pipe with owner-only DACL (aligns with Unix 0o600 semantics)          |

**Key implementation details:**

- Named pipe names: `\\.\pipe\<name>`
- Byte-mode pipes (`PIPE_TYPE_BYTE` / `PIPE_READMODE_BYTE`) for framed I/O
- Read/write timeouts enforced via overlapped I/O with `GetOverlappedResult`
- Timeout reset/clamp behavior tested and documented
- Explicit owner-only DACL: one ACE granting `GENERIC_ALL` to the owner SID

### Async Transport

`AsyncNamedPipeStream` wraps `tokio::net::windows::named_pipe` types for async named pipe I/O:

- `AsyncNamedPipeStream` wraps `NamedPipeClient`/`NamedPipeServer`
- Accept-race hardening: handles `WouldBlock`/`ERROR_PIPE_CONNECTED` in `connect_server`
- `Ok(0)` on write handled to prevent infinite loop on pipe close

### Peer Wiring

`ipcprims-peer` connect/listen paths now use named pipes on Windows:

- `connector.rs`: Windows path creates `NamedPipeStream` instead of returning `Unsupported`
- `listener.rs`: Windows path creates `NamedPipeListener` instead of returning `Unsupported`
- Windows peer tests converted from unsupported stubs to full roundtrip tests

### Async Peer Gate Lifted

Windows async peer API is no longer Unix-gated. `AsyncPeer`, `AsyncPeerListener`, and `async_connect` compile and work on Windows named pipes:

- `async_connector.rs`: Windows async path enabled
- `async_listener.rs`: Windows async path enabled
- `async_peer.rs`: Platform gates removed; `#[cfg(feature = "async")]` only
- `handshake.rs`: Platform gates removed

## Security

### Named Pipe Access Control

Default DACL is owner-only, aligning Windows named pipe access with Unix 0o600 semantics:

- Explicit DACL with one ACE granting `GENERIC_ALL` to the owner SID
- Only the pipe owner (creating process) can connect
- Documented in SDR-0002 alongside the Windows peer identity limitation (no uid/gid equiv)

### Peer Disconnect Handling

`BrokenPipe` and `ConnectionReset` frame errors are now classified as `Disconnected` (not `Fatal`):

- On Windows, named pipe closure surfaces as `BrokenPipe` (OS error 109) rather than EOF
- Previously caused echo server to exit instead of accepting the next connection
- Extracted `classify_frame_error()` as testable free function with cross-platform unit tests

## Developer Experience

New target reports shell type, OS, platform, toolchain versions, and foundation tool availability. Useful for diagnosing missing PATH entries after shell restarts and for CI environment validation.

Closes the host-target clippy gap: on Windows, `make prepush` now also runs a Linux-target clippy pass over Unix-gated code paths that Windows CI would otherwise miss.
